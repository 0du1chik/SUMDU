# Завдання на створення програми:

**Розробити програму, яка читає із файлу БД MySQL інформацію про асортимент дитячих іграшок у магазині, про який відомо:** назва іграшки, ціна, кількість, вікові межі (наприклад, 2–5 років).

**Користувач – адміністратор** може додавати нові записи у файл БД та редагувати наявну інформацію у формі програми. Для доступу до функцій модифікації даних (додавання та редагування) користувач має авторизуватись через пункт головного меню.

**Користувач – оператор** може переглядати у головній формі програми повний перелік усіх наявних записів та виконувати обробку інформації за заданими параметрами:
* отримання назв іграшок, що підходять дітям у віковому діапазоні від **X до Y років**;
* визначення вартості найдешевшої іграшки та її назви.

Значення X та Y вводяться користувачем у відповідні текстові поля головної форми. Результати проведеного відбору інформації користувач може зберегти у файл MS Word, який автоматично формується.

**Обмеження програмного продукту:**
* База даних повинна містити інформацію про не менше ніж 10 різних іграшок;
* Програма повинна забезпечувати цілісність даних при редагуванні записів;
* Вихідний файл MS Word має відповідати визначеній структурі та містити результати обробки.

# Обмеження проєкту

* Тривалість проекту із розроблення ПП – з 16.02.2026 р. до 01.06.2026 р.;
* Програма має бути написана мовою C# у IDE Visual Studio із використанням фреймворку UI [Windows Presentation Foundation (WPF)](https://learn.microsoft.com/en-us/dotnet/desktop/wpf/get-started/create-app-visual-studio);
* Проектування має спиратися на моделі процесів компанії у нотації IDEF+BPMN та моделі програмного продукту відповідно до RUP у нотації UML;
* Реалізація програми буде вестись методом Scrum. Product Backlog буде поділений на 2 спринта тривалістю по 2 тижні;
* По завершенні проєкту програма повинна виконувати всі передбачені функції, мати документацію, перелік якої наведений у розділі Склад програмної документації.
